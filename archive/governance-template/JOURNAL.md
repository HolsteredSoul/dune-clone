# Journal (template)

Append-only development log. Newest entry on top. Terse. Every session APPENDS one entry
before it ends and READS the latest entries + the Drift Log before it starts.

Flow of knowledge: observed here (journal) → ruled in memory/ → if it recurs, promoted to
CONSTITUTION.md as law. This file is the log, not the rulebook — keep durable conclusions in
memory and keep entries short, or this rots into write-only noise.

## Drift Log (read this first)
Every time a session catches itself or a prior session substituting a proxy for the objective,
adding a gate, scope-creeping, or chasing a small sample — one line here, so the next session
knows where the rails are slippery.
